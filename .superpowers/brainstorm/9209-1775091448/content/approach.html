구현 방식 선택

RFID + 사용자 통계를 어느 보드에서 처리할까요?

A

Sensor 보드에 RFID 연결 ★ 추천

MFRC522 → Sensor 보드(SPI) → MQTT로 UID 발행 → Dashboard가 사용자 매핑 처리

✓ 배선 단순 (Sensor 보드 근처에 리더기 설치)
✓ Dashboard가 모든 사용자 데이터 중앙 관리
✓ 플래시 저장도 Dashboard 보드 한 곳에서만
✗ Sensor 보드 코드가 조금 늘어남
B

Dashboard 보드에 RFID 연결

MFRC522 → Dashboard 보드(SPI) → 직접 UID 읽어서 사용자 전환

✓ MQTT 토픽 추가 없음
✗ Dashboard 보드가 HTTP + MQTT + SPI RFID 동시 처리
✗ 리더기가 대시보드 보드 위치에 고정됨
C

별도 4번째 보드로 분리

RFID 전용 Pico를 추가 → MQTT로 UID만 발행

✓ 역할 분리가 가장 깔끔
✗ Pico 보드 1개 추가 필요
✗ 4일 프로젝트에 과함
