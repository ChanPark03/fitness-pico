# 프로젝트 기술서

## 작성자

- HRD 1조
- 이상수, 박찬웅, 홍지나

## 1. 프로젝트명

- `fitness-pico`
- Raspberry Pi Pico 2 W 기반 푸시업 트래커 및 RFID 사용자 통계 시스템

## 2. 수행기간

- 2026.03.30 ~ 2026.04.03 (5일)

## 3. 담당 역할

- 이상수: Dashboard 보드, HTTP API, MQTT 집계, 사용자 데이터 처리
- 박찬웅: Sensor 보드, HC-SR04 기반 푸시업 카운트, RFID UID 발행
- 홍지나: Display 보드, LCD/부저 피드백, 발표 자료 및 시연 정리

## 4. 수행 목표

- Raspberry Pi Pico 2 W 3대를 연동해 센서, 대시보드, 디스플레이가 분리된 IoT 운동 시스템을 구현한다.
- HC-SR04 초음파 센서로 푸시업 반복 수와 세트를 측정하고 MQTT로 실시간 전송한다.
- MFRC522 RFID 리더를 이용해 사용자를 전환하고, 대시보드에서 사용자별 운동 통계를 분리해 확인할 수 있게 한다.
- Pico SDK, 임베디드 C, MQTT, 간단한 웹 UI를 함께 사용해 하드웨어 제어와 소프트웨어 통합 경험을 확보한다.

## 5. 사용 기술

- 언어: C
- 보드: Raspberry Pi Pico 2 W
- 개발 환경: VS Code, CMake, Pico SDK
- 통신: Wi-Fi, MQTT, HTTP, mDNS
- 센서 및 부품: HC-SR04, MFRC522, I2C LCD, Buzzer
- 구현 요소: lwIP MQTT, 내장 HTTP 서버, 플래시 사용자 저장

## 6. 세부 수행 내용

### 6.1 목적

- 보드 3대를 역할별로 분리해 센싱, 제어, 표시를 동시에 수행하는 IoT 운동 시스템을 구축한다.
- 단순 카운트 기능을 넘어서 사용자 식별, 실시간 상태 표시, 웹 제어까지 포함한 통합형 프로젝트를 구현한다.

### 6.2 개발 환경

- 운영 보드: Raspberry Pi Pico 2 W
- 펌웨어: Pico SDK 기반 C 프로젝트
- 빌드 도구: CMake
- 통신 프로토콜: MQTT, HTTP
- 브로커: Mosquitto
- 개발 도구: VS Code

### 6.3 주요 기능

- 푸시업 거리 측정 후 반복 수와 세트 수 계산
- 반복 속도 측정 및 빠름/느림 경고 표시
- Dashboard 웹 UI에서 운동 시작/종료 제어
- 보드 heartbeat를 이용한 연결 상태 표시
- RFID 카드 태그로 사용자 전환
- 대시보드에서 RFID 사용자 등록 및 현재 사용자 통계 표시

## 7. 구성도

프로젝트 전체 흐름은 아래 자산을 기준으로 설명할 수 있다.

- 워크플로우 이미지: [assets/fitness-pico-workflow.svg](assets/fitness-pico-workflow.svg)
- 아키텍처 설명 문서: [architecture.md](architecture.md)

구성 요약:

1. Sensor board가 HC-SR04와 MFRC522를 읽고 MQTT로 운동 상태와 RFID UID를 발행한다.
2. Dashboard board가 MQTT 메시지를 수신해 상태를 집계하고 브라우저용 HTTP UI를 제공한다.
3. Display board가 반복 수, 속도, 사용자 전환 상태를 LCD와 부저로 표시한다.
4. Browser는 Dashboard board에 접속해 운동 제어와 RFID 등록을 수행한다.

## 8. 데이터 저장 구조

이 프로젝트는 별도 DB 서버를 사용하지 않고 Dashboard 보드 플래시에 사용자 정보를 저장한다.

### 8.1 사용자 저장 구조

| 항목 | 설명 |
| --- | --- |
| `uid` | RFID 카드 UID |
| `name` | 사용자 이름 |
| `weight_kg` | 체중 |
| `goal_sets` | 목표 세트 수 |

### 8.2 통계 저장 방식

| 항목 | 저장 위치 | 설명 |
| --- | --- | --- |
| 사용자 매핑 | Dashboard 보드 플래시 | 재부팅 후에도 유지 |
| 현재 사용자 통계 | Dashboard 보드 RAM | 런타임 기준 누적 값 |

## 9. 플로우 차트

### 9.1 운동 측정 흐름

1. 브라우저에서 운동 시작 요청
2. Dashboard board가 `fitpico/control` 발행
3. Sensor board가 추적 시작
4. HC-SR04 값으로 푸시업 반복 수 계산
5. MQTT로 count/speed/rest/daily 상태 발행
6. Dashboard와 Display가 상태를 갱신

### 9.2 RFID 사용자 흐름

1. 카드 태그
2. Sensor board가 UID를 읽어 `fitpico/rfid/uid` 발행
3. Dashboard board가 등록 여부 확인
4. 등록된 카드면 현재 사용자를 전환
5. 미등록 카드면 등록 대기 상태로 표시
6. 등록 완료 후 사용자 정보는 플래시에 저장
